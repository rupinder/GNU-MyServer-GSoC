PAGINE DI ERRORE



I files di default sono contenuti nella cartella system del virtual host e possono essere rimpiazzati con i tuoi. Differenti Virtual Hosts possono usare differenti files di errore.

La pagina non deve contenere riferimenti ad altri files nella cartella system.


Il famoso errore 404 avviene quando il server non trova il file sul server.


Per definire una pagina di errore personalizzata per un virtual host puoi aggiungere una linea come la seguente nel primo file security del virtual host(il file security presente nella directory web del virtual host). Se vengono definiti files personalizzati i files verrano cercati nella cartella del web invece che in quella system.
I files cercati nella cartella system devono avere un nome x.html dove x è l'errore HTTP da inviare al client. Se neanche questo file viene trovato il server invierà al client l'errore solo attraverso il codice di errore HTTP. Con questa riga si configura il web server per inviare il file error404.html in caso di errore 404.

<ERROR FILE="error404.html" ID="404" />



Per maggiori informazioni sugli errori HTTP controlla il documento RFC2068 a:www.w3c.org.
