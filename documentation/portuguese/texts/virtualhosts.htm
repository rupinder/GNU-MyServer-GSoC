HOSTS VIRTUAIS

Na teoria, virtual hosts permitem aos usuários colocar um número infinito de sites em uma única máquina. Por exemplo http://mypc.net é diferente de http://sub.mypc.net. Para definir virtual hosts no myServer, abra o arquivo virtualhosts.xml em um editor de texto (como o notepad no Windows ou o Emacs no Linux). Uma configuração de demonstração do arquivo de hosts virtuais se parece com isso:

<VHOSTS>
-
    <VHOST>
<NAME>Todas as conexões</NAME>
<PORT>80</PORT>
<PROTOCOL>HTTP</PROTOCOL>
<DOCROOT>web</DOCROOT>
<SYSFOLDER>system</SYSFOLDER>
<ACCESSLOG>logs/myServer.log</ACCESSLOG>
<WARNINGLOG>logs/myServer.err</WARNINGLOG>
</VHOST>
</VHOSTS>

<NAME>  dá ao administrador do sistema uma idéia do que esse virtual host é, se está só na Ethernet interna, na externa, ou para todas as conexões.

<PORT> é a porta usada para conexões aceitas
<PROTOCOL>  é o protocolo usado pelo host
<DOCROOT> é a pasta onde os arquivos se localizam
<SYSFOLDER> é a pasta system
<ACCESSLOG> é o arquivo de log de acessos usado pelo virtual host
<WARNINGLOG> é o arquivo de log de erros usado pelo virtual host.

A flag de logs também pode vir dessa forma:
<ACCESSLOG type="combined">logs/myServerHTTP.log</ACCESSLOG>
Com essa opção, os logs terão dois campos extras: HTTP Referer e HTTP User Agent.
Usando cycle="yes" você habilita o log-rotate do arquivo de log. Quando um arquivo de log com cycling habilitado alcançar o tamanho máximo para arquivos de log, ele vai escrever seu conteúdo em um novo arquivo e recriar o arquivo original. Por default os arquivos de log ciclados usam compactação gzip para economizar espaço, se você não quiser usar gzip, especifique cycle_gzip="no".

O MyServer lhe dá a opção de habilitar o comando HTTP TRACE para cada Host Virtual. Tudo o que você precisa fazer é editar o arquivo security.xml do diretório raiz do seu host (não de dentro de um subdiretório dele! Se o arquivo não existir, você pode criar um com o formato descrito na seção de Segurança, dessa documentação), e adicione o parâmetro

<HTTP TRACE="ON" />

Uma vez feito aquilo, você terá habilitado o HTTP TRACE. Por padrão, ele vem desabilitado (ou seja, apenas apague a linha, caso deseje desligar essa opção)
