SERVEUR WEB

Qu'est ce qu'un serveur web?

Un serveur web est un programme permettant aux utilisateurs de publier des documents sur Internet ou sur un réseau local.

Vos visiteurs ont uniquement besoin d'un navigateur pour se connecter à et accéder aux pages de votre serveur web.


Plusieurs clients se connectant à un serveur web.

Cette image montre comment des clients variés peuvent accéder à votre serveur au même moment. Lorsqu'un client demande à consulter une de vos page, cette requête arrive sur votre serveur grâce au réseau. Une réponse est générée par le serveur et renvoyée au client. L'évolution d'Internet a fait du modèle Client-Serveur la façon la plus simple de partager rapidement des données, offrant ainsi la possibilité à chaque utilisateur de choisir ce qu'il ou elle veut partager avec les autres utilisateurs.

Comment les fichiers sont-ils partagés?

Grâce au niveau le plus basique, le serveur web transmet des documents HTML et des images. Ces documents et images sont de simples fichiers statiques correspondant aux fichiers que vous voyez lorsque vous accédez à un site web. Ces fichiers sont transférés du serveur au client par le protocole HTTP (HyperText Transfer Protocol).

Si du contenu dynamique est nécessaire, par exemple pour faire fonctionner un site d'e-commerce, différents protocoles sont utilisés pour la communication du serveur web avec un processus externe. On trouve parmis ces protocoles : Common Gateway Interface (CGI), ISAPI, FastCGI ou bien encore WinCGI.
HTTP n'est pas un protocole sécurisé, c'est à dire que les données transmises ne sont pas cryptées. Le cryptage reste possible pour des transmissions sécurisées grâce à HTTPS (HyperText Transfer Procol Secure). Plus d'informations sur HTTPS peuvent être trouvées ici.

Traduction de chemin

Les serveurs web traduisent un chemin d'une URI(identifiant uniforme de ressource) en une ressource système locale. Le chemin URI spécifié par le client est relatif au répertoire racine du serveur web (Hôtes virtuels). Par exemple, le client peut demander l'accès au fichier /mydirectory/myfile avec la commande suivante:

GET /mydirectory/myfile HTTP/1.1
Host: www.host.com

Le serveur rajoute au nom du fichier le répertoire racine de l'hôte virtuel. Si le répertoire racine est /var/www, alors la ressource retournée sera /var/www/mydirectory/myfile.
