HÔTES VIRTUELS


En théorie, les hôtes virtuels permettent d'héberger un nombre infini de sites sur une même machine. Par exemple, l'adresse http://mypc.net est différente de http://sub.mypc.net. Pour définir des hôtes virtuels dans MyServer, ouvrez le fichier virtualhosts.xml dans un éditeur de texte (comme le Bloc Notes ou GNU Emacs sous GNU/Linux). Un fichier de configuration d'hôte virtuel ressemble à ceci:

<VHOSTS>
-
<VHOST>
<NAME>Every connection</NAME>
<PORT>80</PORT>
<IP>127.0.0.1</IP>
<PROTOCOL>HTTP</PROTOCOL>
<DOCROOT>web</DOCROOT>
<SYSFOLDER>system</SYSFOLDER>
<HOST>www.mysite.com</HOST>
<ACCESSESLOG>logs/myServer.log</ACCESSESLOG>
<WARNINGLOG>logs/myServer.err</WARNINGLOG>
<ALLOW_CGI>NO</ALLOW_CGI>
</VHOST>
-
</VHOSTS>




<NAME> indique à l'administrateur si l'hôte est accessible en interne, en externe ou les deux. 
<PORT> est le port utilisé pour les connexions.
<IP> est optionnel. Il s'agit de l'adresse IP de l'interface à utiliser pour l'hôte virtuel. La propriété optionnelle useRegex="YES" indique que l'adresse est une expression régulière, et non pas une adresse fixe. Cela est plus pratique pour des réseaux à adressage d'IP dynamique.
<PROTOCOL> est le protocole à utiliser par l'hôte.
<DOCROOT> est le répertoire racine de l'hôte où seront placés les fichiers.
<SYSFOLDER> est le répertoire système.
<HOST> est le nom de votre hôte virtuel. La propriété optionnelle useRegex="YES" indique que l'adresse est une expression régulière, et non pas un nom fixe.
<ACCESSLOG> est le registre d'accès utilisé par l'hôte virtuel.
<WARNINGLOG> est le registre d'erreurs utilisé par l'hôte virtuel.
<ALLOW_CGI> Cette option refusera l'accès au CGI dans l'hôte virtuel. Elle peut égalemnt exister dans le fichier myserver.xml, agissant comme valeur globale, mais la valeur de virtualhosts.xml est prioritaire. ALLOW_CGI peut aussi aparraitre comme ALLOW_FASTCGI, ALLOW_ISAPI, ALLOW_MSCGI, ALLOW_WINCGI, ALLOW_SEND_LINK, ALLOW_EXTERNAL_COMMANDS, ALLOW_SEND_FILE, chacun d'eux autorisant/refusant l'utilisation du gestionnaire de données HTTP.


Les registres peuvent avoir d'autres options:
<ACCESSESLOG type="combined">logs/myServerHTTP.log</ACCESSESLOG>
Cette option ajoute deux champs aux registres : HTTP Referer et HTTP User Agent.
Utiliser cycle="yes" autorisera le log en boucle. Lorsqu'un registre est cyrculaire, il recopie son contenu dans un nouveau fichier lorsqu'il atteint une taille maximale avant de se vider. Les fichiers créés sont compressés en gzip afin de gagner de la place. Vous pouvez annuler cela avec l'option cycle_gzip="no".

MyServer vous permet d'autoriser la commande HTTP TRACE pour chaque hôte virtuel. Il suffit pour cela d'éditer le fichier security.xml de la racine de votre hôte virtuel (sans sous répertoire! Vous pouvez créer le fichier en vous référant à la section Sécurité de cette documentation), en ajoutant le paramètre

<HTTP TRACE="ON" />

Une fois effectué, la commande HTTP TRACE sera disponible. Par défaut, elle n'est pas activée.



MyServer offre la possibilité de suivre les liens symboliques sur des systèmes d'exploitation de type Unix (i.e.: Linux, *BSD, GNU/HURD, Solaris, etc.).
Vous pouvez autoriser ce fonctionnement pour chaque hôte virtuel en modifiant le fichier virtualhosts.xml.
<FOLLOW_LINKS>YES</FOLLOW_LINKS>

La valeur utilisée par défaut dans MyServer est : <FOLLOW_LINKS>NO</FOLLOW_LINKS>
