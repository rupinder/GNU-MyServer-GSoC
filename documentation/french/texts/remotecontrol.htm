contrôle DISTANT

MyServer permet la gestion distante de sa configuration, via le Centre de Contrôle, depuis n'importe où dans le monde. Vous avez juste besoin d'autoriser le protocole de contrôle.

Pour cela, vous devez ajouter un hôte virtuel qui utilisera le protocole de contrôle. Vous pouvez faire cela en modifiant le fichier virtualhosts.xml, de manière à le faire ressembler à ceci:

<VHOST>
<NAME>Every control connection</NAME>
<PORT>270</PORT>
<SSL_PRIVATEKEY>certificates/server.key</SSL_PRIVATEKEY>  
<SSL_CERTIFICATE>certificates/server.pem</SSL_CERTIFICATE >  
<PROTOCOL>CONTROL</PROTOCOL>
<DOCROOT></DOCROOT>
<SYSFOLDER></SYSFOLDER>
<ACCESSLOG>logs/control.log</ACCESSLOG>
<WARNINGLOG>logs/control.err</WARNINGLOG>
</VHOST> 

La ligne <PORT>270</PORT> identifie le port à utiliser pour l'accès au contrôle distant. Dans l'exemple ci dessus, vous pourriez accéder au contrôle distant avec l'interface en vous connectant au port 270.

Notez que les fichiers certificates/server.key et certificates/server.pem doivent être créés pour pouvoir utiliser ce service. Référez vous au manuel de création de clés SSL pour plus d'informations à ce sujet.

Vous devez également modifier le fichier myserver.xml pour définir un mot de passe d'administration, en ajoutant les lignes:

<CONTROL_ENABLED>YES</CONTROL_ENABLED> 
<CONTROL_ADMIN>ADMIN</CONTROL_ADMIN> 
<CONTROL_PASSWORD>PASSWORD</CONTROL_PASSWORD> 

Changez juste les valeurs de ADMIN et PASSWORD pour les faire correspondre au mot de passe et login de l'administrateur, et souvenez vous de mettre à YES le paramètre CONTROL_ENABLED (sinon vous n'aurez pas accès au contrôle distant).

N'oubliez pas de définir un nom et un mot de passe pour l'administrateur, de préférence en n'utilisant pas des mots simples comme admin ou password.
